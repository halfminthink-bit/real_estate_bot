## 執筆ガイドライン

### 1. 文章量
**全体で3000-5000文字**
- セクション1（フック）: 短め
- セクション2（データ）: 長め
- セクション3（理由）: 中程度
- セクション4（CTA）: 短め

### 2. 文章スタイル

✅ **DO（推奨）**
- 友達に教えるような親しみやすさ
- 1文は50文字以内
- シンプルに、短く
- 数値は具体的に（「高い」→「+21%」）
- 問いかけを使う

❌ **DON'T（禁止）**
- 説教的な表現
- 同じ表現の繰り返し（2回まで）
- 専門用語の羅列
- 長い文章
- HTML/CSS/Markdown表の出力

### 3. 特殊マーカーの配置（重要）

以下の4つのマーカーを**正確な位置**に配置してください：

#### <CHART>
- **位置**: セクション2（2番目の##）の見出し直後
- **前後**: 必ず空行

**例**:
```
## セクション2の見出し

<CHART>

グラフをご覧ください...
```

#### <DATA_TABLE>
- **位置**: セクション2のグラフ説明の後
- **内容**: 最新年度のデータ（平均地価、価格帯、地点数、変動率）
- **前後**: 必ず空行

**例**:
```
...26年間の推移です。

<DATA_TABLE>

（出典：国土交通省 地価公示データ {latest_year}年）

注目すべきは...
```

#### <TRANSACTION_DATA>
- **位置**: セクション2のデータ表（<DATA_TABLE>）の後
- **内容**: 取引価格の統計と代表的な取引事例（2-3件）
- **前後**: 必ず空行
- **条件**: has_transaction_dataがTrueの場合のみ配置

**例**:
```
（出典：国土交通省 地価公示データ {latest_year}年）

<TRANSACTION_DATA>

では、実際にはどのような価格で取引されているのでしょうか...
```

**注意事項**:
- 「{area_name}エリア全体」の取引であることを必ず明記
- 「{choume}を含む周辺エリアの参考値」として扱う
- 取引期間: {transaction_period}（過去{transaction_years}年分）を明記
- 取引件数: {transaction_count}件を強調
- has_transaction_dataがFalseの場合は、このマーカーを配置しない
- マーカーの前後には必ず空行を入れる

**記事での表現例**:
✅ 良い例:
「過去3年間（2022年〜2024年）に、{area_name}エリア全体で{transaction_count}件の土地取引が記録されています。」
「この3年間のデータを見ると、平均取引価格は{transaction_avg:,}円ですが、価格帯には大きな幅があります。」

❌ 悪い例:
「{transaction_year}年第{transaction_quarter}四半期に{transaction_count}件の取引」（期間が短すぎる印象）
「最近の取引では」（曖昧な表現）

**禁止事項**:
- `![]()`形式の画像タグ
- `| 項目 | データ |`形式のMarkdown表
- HTML/CSSタグ
- マーカー以外の装飾

#### <AFFILIATE>
- **位置**: セクション4の最後
- **前後**: 必ず空行

**例**:
```
...知ってみませんか？

<AFFILIATE>

まずは知ることから。
```

**禁止事項**:
- `![]()`形式の画像タグ
- `| 項目 | データ |`形式のMarkdown表
- HTML/CSSタグ
- マーカー以外の装飾

### 4. データの柔軟な解釈

データに応じて、最も説得力のあるストーリーを構築してください：

**価格帯の幅がある場合**:
「同じ町内でも○倍の差」→ 査定の必然性

**上昇トレンド**:
「市場の評価」「リーマン・コロナでも底堅い」

**減少トレンド**:
「だからこそ知るべき」「現状把握の重要性」

**横ばい**:
「安定した価値」「底堅さの証明」

### 5. 記事構成（3セクション）

#### セクション1: フック + 信頼性
1. 問題提起（読者の悩み）
2. 筆者の立場（独立系アナリスト）
3. データの信頼性（国土交通省、26年分）

#### セクション2: データ分析
1. <CHART>配置
2. グラフの説明（推移、主要イベント）

【グラフの詳細分析（最重要・必須）】

land_price_history_detailには、各年の価格と前年比変動率が含まれています。
以下の観点で必ず分析してください：

**1. リーマンショック（2008-2009年）の影響**
- lehman_changeを確認してください
- マイナス5%以上の下落があった場合: 「リーマンショックで○○%下落」と明記
- マイナス5%未満の場合: 「リーマンショックの影響は軽微」

**2. コロナ禍（2020-2021年）の影響**
- covid_changeを確認してください
- マイナス5%以上の下落があった場合: 「コロナ禍で○○%下落」と明記
- マイナス5%未満の場合: 「コロナ禍でも価格は安定」

**3. 長期トレンドの正確な表現**
以下の基準で表現を選択してください：

✅ 「緩やかな上昇」と書ける条件:
- 各年の前年比変動率がすべてマイナス5%以内
- price_change_26yがプラス

✅ 「V字回復」と書ける条件:
- リーマンまたはコロナで大きく下落（-5%以上）
- その後、回復してプラスになっている

✅ 「安定的に推移」と書ける条件:
- 各年の変動率が-5%〜+5%の範囲内

❌ 禁止表現（データを見ずに書いてはいけない）:
- 「緩やかな上昇トレンドを保ち続けています」← データ確認なし
- 「安定した上昇」← 大きな変動があった場合は不正確

**4. 最終的な変動率の正確な解釈**
例: price_change_26y = +1.2%の場合

❌ 悪い例: 「26年間で+1.2%の上昇を続けています」
✅ 良い例: 「リーマンショックやコロナ禍で一時的に下落したものの、長期的には+1.2%の上昇となっています」

**5. land_price_history_detailの活用**
年次データの具体例を1-2箇所引用してください：
例: 「2008年には○○万円/㎡でしたが、リーマンショック後の2009年には○○万円/㎡（-○○%）まで下落しました」

3. <DATA_TABLE>配置
4. 出典明記
5. データ解説（価格帯の幅、変動率）

【価格の乖離を強調する（最重要）】

has_wide_range=Trueの場合、以下を必ず含めてください：

1. 倍率の明示
「同じ{choume}でも、最高値と最安値では約{price_ratio}倍の価格差が生じています」

2. 損失の可能性
「平均価格だけを見て『うちはこれくらいか』と判断するのは、数千万円単位の損をしている可能性があります」

3. 読者への問いかけ
「あなたの土地は、この価格帯のどこに位置するのでしょうか？」

4. 査定の必然性
「個別の条件によって価格は大きく変わります。だからこそ、正確な査定が必要なのです」

※<DATA_TABLE>の直後（出典の後）に配置してください

**重要**: <DATA_TABLE>マーカーで表が挿入されるため、以下のような重複セクションは作成しないでください：
- ❌ 「【最新データ（{latest_year}年）】」という見出しとデータリスト
- ❌ 「【参考：{oldest_year}年時点（推定）】」というセクション
- ❌ 表と同じ情報をテキストで繰り返す

表に既に含まれている情報（平均地価、価格帯、変動率）を再度テキストで記載する必要はありません。

#### セクション3: なぜ知るべきか
4つの理由を簡潔に：
1. 相続対策
2. 資産の見える化
3. 将来の選択肢
4. 金融機関との交渉

締めのメッセージ:
「今すぐ売る必要はない。でも知っておくべき」

#### セクション4: CTA
1. 問いかけ（価格帯の幅）
2. 行動への誘導
3. <AFFILIATE>配置
4. 最後の一押し（「まずは知ることから」程度で簡潔に）

### 7. 取引価格データの扱い方

#### has_transaction_dataがTrueの場合

✅ **DO（推奨表現）**
- 「過去3年間（{transaction_period}）に、{area_name}エリア全体で{transaction_count}件の土地取引が記録されています」
- 「{choume}を含む周辺エリアの参考値として、実際の市場動向を見てみましょう」
- 「地価公示データが『評価額』を示すのに対し、こちらは『実際に売買された価格』です」
- 「長期的な価値（地価公示）と、過去3年間の市場動向（取引価格）の両面から見ることができます」
- 「この3年間のデータを見ると、平均取引価格は{transaction_avg:,}円ですが、価格帯には大きな幅があります」

❌ **DON'T（禁止表現）**
- 「{choume}では...」← エリア全体なので不正確
- 「この地点で...」← 地点特定できない
- 「あなたの土地と同じ条件で...」← 誤解を招く

#### has_transaction_dataがFalseの場合

✅ **DO（推奨）**
- セクション2は地価公示データのみで構成
- <TRANSACTION_DATA>マーカーは配置しない
- 「地価公示データから見る長期的な価値」に焦点を当てる

#### 取引事例の紹介方法

**良い例**:
```
【{area_name}エリアの取引事例（{transaction_period}）】

過去3年間（{transaction_period}）に、{area_name}エリア全体で{transaction_count}件の土地取引が記録されています。

代表的な取引事例：

1. 中古マンション等: 3,500万円
   - 建築年: 1968年
   - 間取り: ３ＬＤＫ
   - 用途地域: 第一種低層住居専用地域
   - 取引時期: 2024年第3四半期

2. 中古マンション等: 3,700万円
   - 建築年: 2003年
   - 間取り: １ＬＤＫ
   - 取引時期: 2023年第2四半期

（注：{choume}を含む{area_name}エリア全体の取引データです）
```

**悪い例**:
```
❌ あなたの土地も同じ価格で売れます
❌ {choume}の取引価格は3,500万円です
❌ この地点の市場価格は...
```
